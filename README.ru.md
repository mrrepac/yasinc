# yasinc

Удобная двусторонняя синхронизация хранилища Obsidian с **Яндекс.Диском** через
официальный REST API. Работает на десктопе и на телефоне (всё через
`requestUrl`, без сторонних серверов). Сравнение файлов — по **sha256**,
конфликты не затирают данные, удаления уходят в корзину.

## Что делает

- **Двусторонняя синхра** папки хранилища с папкой на Яндекс.Диске.
- **Трёхстороннее сравнение** (локально ↔ снимок прошлой синхры ↔ Диск) —
  корректно различает «файл удалён» и «файл появился на другой стороне», не
  воскрешает удалённое.
- **Ручной и автоматический режим**: кнопка на ленте, команда с хоткеем,
  синхра при запуске и по таймеру.
- **Безопасность прежде всего**: при расхождении с двух сторон создаётся
  `.conflict-…`-копия (ничего не теряется); удаления — в корзину Диска и
  системную; после скачивания проверяется контрольная сумма.

## Настройка (один раз)

### 1. Зарегистрировать приложение Яндекса

1. Открой <https://oauth.yandex.ru/client/new> (под своим аккаунтом Яндекса).
2. Название — любое (например `yasinc`).
3. Платформы → включи **«Веб-сервисы»**, в Redirect URI вставь:
   `https://oauth.yandex.ru/verification_code`
4. Доступы → добавь права Яндекс.Диска:
   `cloud_api:disk.read`, `cloud_api:disk.write`, `cloud_api:disk.info`
5. Создай приложение и скопируй **ClientID**.

### 2. Получить токен в плагине

1. Настройки → yasinc → вставь **ClientID**.
2. Нажми **«Открыть страницу авторизации»** → **«Разрешить»**.
3. Тебя перекинет на страницу с адресом вида
   `…/verification_code#access_token=…`. Скопируй весь адрес (или сам токен) и
   вставь в поле **«Токен доступа»** — плагин вытащит токен сам.
4. Нажми **«Проверить подключение»** — должно показать свободное место.

Токен и снимок синхры хранятся только локально (в `data.json` и
`sync-state.json` плагина) и **никогда не уезжают на Диск**.

## Настройки

| Параметр | Смысл |
|---|---|
| **Папка на Диске** | Куда складывать хранилище (по умолчанию `Obsidian/<имя хранилища>`). |
| **Синхронизировать `.obsidian`** | Настройки, темы и плагины тоже едут между устройствами. |
| **Синхронизировать при запуске** | Автосинхра через несколько секунд после старта. |
| **Автосинхронизация** | Фоновая полная сверка по таймеру (5–60 минут или выкл). |
| **Автозаливка правок** | Через N секунд тишины после правки изменённые файлы сами улетают на Диск (инкрементально, без полного скана). |
| **Исключения** | Что не синхронизировать (по шаблону на строку). |
| **Сбросить состояние синхры** | Забыть снимок; следующий запуск сольёт стороны заново, ничего не удаляя. |

### Встроенные исключения

Всегда исключены и в списке не показываются: `.git`, `node_modules`, `.trash`,
`.DS_Store`, `Thumbs.db`, workspace-файлы Obsidian, `cursor-positions*`
(remember-cursor-position) и папка самого yasinc (снимок и кеш синхры).
Поле «Исключения» — только для твоих собственных шаблонов.

Правила шаблонов:
- **Без `/`** — совпадает с любой папкой/файлом с таким именем в любом месте
  (`node_modules`, `.git`, `*.tmp`).
- **Со `/`** — путь от корня хранилища (`.obsidian/workspace.json`).

## Как разрешаются ситуации

| Что произошло | Что делает yasinc |
|---|---|
| Файл изменён только локально | Заливает на Диск |
| Файл изменён только на Диске | Скачивает |
| Файл изменён и там, и там | Оставляет твой локальный + кладёт удалённый рядом как `имя.conflict-ГГГГММДД-ЧЧмм.ext`. Обе версии сохранены с обеих сторон |
| Файл удалён локально | Удаляет на Диске (в корзину) |
| Файл удалён на Диске | Удаляет локально (в корзину) |
| Файл удалён на одной стороне, но изменён на другой | Восстанавливает (правки важнее удаления) |

**Предохранитель:** если при синхре одна из сторон внезапно оказалась пустой
(а раньше была не пуста) — синхра отменяется, чтобы случайный сбой чтения не
удалил данные на другой стороне.

## Использование

- Иконка ⟳ на ленте — синхронизировать сейчас.
- Команда **«yasinc: Синхронизировать сейчас»** (Ctrl+S по умолчанию). После
  первой полной синхры сессии работает инкрементально — улетает только
  изменённое, за доли секунды.
- Команда **«yasinc: Остановить синхронизацию»** — прервать после текущего файла.
- Команда **«yasinc: Журнал последней синхронизации»** — список: что залито,
  скачано, удалено, где конфликты (также кнопка-список в настройках).
- Клик по индикатору в статусбаре.
- Индикатор показывает сводку: `↑` залито, `↓` скачано, `🗑` удалено,
  `⚠` конфликтов, `✖` ошибок, `✎N` — несинхронизированных правок; при
  наведении во время синхры — какой файл сейчас обрабатывается.
